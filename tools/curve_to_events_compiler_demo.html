What This Demonstrates

This is not a lerp replacement. Lerp moves continuous values. This realizes countable things: drops, spawns, pixels, frames, jobs, particles, tiles, resource transfers.

The impressive part is that the green method follows a changing shape while keeping cumulative error under one event. It is basically a tiny deterministic compiler from “desired probability/density” into “actual discrete outcomes.”

Try high target events with Pulse Train, then switch to Pixel Movement Race. The random runner wanders. The fixed runner is smooth but wrong for shaped targets. The cumulative runner follows the target and lands exactly.

Built as a single-file demo. No libraries. No network. Core: u[t] = floor(C[t+1]) - floor(C[t]).
